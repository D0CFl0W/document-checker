# Каталог `services/` — ответственный за логику

Здесь лежит вся **бизнес-логика** вне HTTP-слоя: распаковка архива, извлечение текста из DOCX/PDF, извлечение полей по шаблонам, генерация PDF-отчёта.

## Файлы

### `unpacker.py`

| Функция | Назначение |
|---------|------------|
| `extract_to_isolated_dir(archive_path, target_base_dir)` | Проверяет, что архив — файл; создаёт `target_base_dir / <stem архива>`; распаковывает туда ZIP; при ошибке удаляет созданную папку. |
| `build_packaged_report(archive_path, unpack_base_dir, report_basename)` | Распаковка → `parse_directory` → `generate_pdf_report` → путь `reports/<report_basename>.pdf`. |

Импортируйте из `services.parser` **явные** символы (`parse_directory`, `generate_pdf_report`), не используйте `import *`.

### `parser.py`

| Функция | Назначение |
|---------|------------|
| `_detect_format(path)` | Внутренняя: по первым байтам (`%PDF`, ZIP/`PK`) и при необходимости по расширению выбирает `.pdf` или `.docx`. |
| `extract_pdf` / `extract_docx` | Сырой текст из файла. |
| `extract_text` | Унифицированное чтение по результату `_detect_format`. |
| `normalize_text` | Подготовка текста к регэкспам (пробелы, переносы). |
| `parse_document` | Один файл → словарь полей (`ФИО`, `Группа`, `Тема`, `Дата`, `Подпись`). |
| `parse_directory` | Все `*.docx` и `*.pdf` в каталоге (по умолчанию до `limit=5` файлов), сортировка по имени; на каждый файл либо `{"file", "data"}`, либо `{"file", "error"}`. |
| `generate_pdf_report` | ReportLab: многостраничный PDF с кириллицей через `DejaVuSans.ttf`. |

Константа **`patterns`**: списки регэкспов с именованной группой `(?P<value>...)`. Для **«Тема»** дополнительно используется эвристика `extract_topic` по строкам с «по теме».

### `__init__.py`

Пакет-маркер; при необходимости можно экспортировать публичный API пакета.

## Шрифты и ресурсы

- `FONT_PATH` указывает на `fonts/DejaVuSans.ttf` относительно корня проекта.
- Без файла шрифта регистрация в ReportLab упадёт при генерации отчёта.

## Ограничения парсера

- Поддерживаются только **DOCX** (OOXML) и **PDF** с извлекаемым текстом (не сканы без OCR).
- Порядок файлов в отчёте задаётся **лексикографической сортировкой** имён в папке после распаковки — имена в ZIP (`1_title...`, `2_task...`) как раз дают предсказуемый порядок.
- Ошибка чтения одного файла **не** останавливает остальные: в списке результатов будет элемент с `"error"`.

## Типичные изменения

- **Новое поле документа:** добавьте ключ и список паттернов в `patterns`, при необходимости — отдельную функцию извлечения и ветку в `parse_document`.
- **Другой формат отчёта:** менять `generate_pdf_report` (или вынести генерацию в отдельный модуль и вызывать из `build_packaged_report`).
- **Вложенные папки в ZIP:** текущий `parse_directory` смотрит только **корень** распаковки; рекурсивный обход потребует правки `glob` или обхода дерева.

## Зависимости модуля

- `python-docx` — DOCX.
- `pdfplumber` (и транзитивно pdfminer и др.) — PDF.
- `reportlab` — выходной PDF.
